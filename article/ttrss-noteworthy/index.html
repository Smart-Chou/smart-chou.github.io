如何正确的配置 Fever Emulation API

通常，为了适配第三方 RSS 阅读器比如 Reeder，我们需要使用 Fever Emulation API 进 行「登录」。首先我们必须开启 Tiny Tiny RSS 外部 API 访问的权限。在 Preference » General » Enable API 处开启：

Fever Emulation 的 API 包含有三个部分：

  • 你的 Fever API 地址（通常是你的服务器域名 + /plugins/fever/）
  • 你的登录用户名
  • 你的 Fever API 密码

在 Preferences » Personal data / Authentication » Personal data » Full name 处设 置的用户名就是你 Fever API 的用户名：

在 Preferences » Fever Emulation 处你可以找到：

  1. Fever API 地址
  2. 设置 Fever API 密码的地方

要知道，设置 Fever API 的密码时点击 Set Password 并不会提示「成功」，不过你可以 从 XHR 请求中看到 Password saved 的 response，所以其实你的密码已经保存啦。用 这里设置的 API 地址、用户名和密码，我们就能成功登录 Reeder 等 RSS 阅读器了。
